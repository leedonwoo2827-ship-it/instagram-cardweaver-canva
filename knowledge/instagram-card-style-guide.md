# Instagram 카드 스타일 가이드 — 1080×1350

Instagram 캐러셀의 4:5 포트레이트(1080×1350)를 기본 비율로 사용한다. Meta 공식 권장이자 피드 점유 면적이 가장 큼.

---

## 캔버스

- **해상도**: 1080×1350 (4:5, 포트레이트)
- **Safe zone**: 상하좌우 64px 여백
- **Bleed**: 상단·하단 160px 구간에는 핵심 텍스트 배치 금지 (피드에서 헤더/하단 UI에 가려질 수 있음)

## 타이포그래피

| 용도 | 폰트 | 크기 | 굵기 | 행간 |
|---|---|---|---|---|
| series_title (우상단 오버라인) | Noto Sans KR | 28pt | 500 | 1.2 |
| headline | Noto Sans KR | 72pt | 700 (Bold) | 1.15 |
| body | Noto Sans KR | 36pt | 400 | 1.45 |
| code (code-card 전용) | JetBrains Mono | 28pt | 500 | 1.5 |
| index_label (우상단) | Noto Sans KR | 32pt | 500 | 1.0 |
| handle/logo (우하단) | Noto Sans KR | 28pt | 500 | 1.0 |

**폰트 라이선스**: Noto Sans KR = SIL Open Font License, JetBrains Mono = Apache 2.0. 템플릿에 @font-face 임베드 권장 (Puppeteer 렌더 시 네트워크 의존 제거).

## 팔레트 프리셋

| 프리셋 | primary | accent | 소스 기본값 |
|---|---|---|---|
| `sepia-cream` | `#3B2E22` | `#E3A857` | book |
| `navy-orange` | `#1E2A4A` | `#F28B3C` | curriculum |
| `dark-cyan` | `#0F1419` | `#22D3EE` | github |

**규칙**: 한 시리즈 = 한 팔레트. 카드별 `palette_override` 는 예외(강조 카드 1장만).

## 레이아웃 — typo-minimal

```
┌──────────────────────────────────────┐
│  [series_title]         [index_label]│
│                                      │
│                                      │
│                                      │
│   [headline — 1~3 줄, 70% 폭]        │
│                                      │
│                                      │
│   [body — 2~4 줄, 80% 폭]            │
│                                      │
│                                      │
│                                      │
│  [logo/handle]                       │
└──────────────────────────────────────┘
   배경: primary,  글자: accent + 흰색
```

- 상단 20% 오버라인 영역
- 중앙 60% 헤드라인+본문 영역
- 하단 20% 로고 영역

## 레이아웃 — photo-overlay

```
┌──────────────────────────────────────┐
│ [FlowGenie 배경 이미지 full-bleed]   │
│                                      │
│ ░░░░ 상단 40% 그라데이션(primary→투명)│
│ [series_title]         [index_label]│
│                                      │
│                                      │
│                                      │
│                                      │
│ ░░░░ 하단 60% 그라데이션(투명→primary)│
│                                      │
│   [headline]                         │
│   [body]                             │
│  [logo/handle]                       │
└──────────────────────────────────────┘
```

- FlowGenie 결과 이미지는 1080×1350 비율로 자동 맞춤(object-fit: cover)
- 상·하단 그라데이션으로 가독성 확보

## 레이아웃 — code-card

```
┌──────────────────────────────────────┐
│  [series_title]         [index_label]│
│                                      │
│   [caption — 8자 라벨]               │
│   ┌────────────────────────────┐     │
│   │ 01  def hello():           │     │
│   │ 02      print('hi')        │     │
│   │ ...                        │     │
│   │ 08                         │     │
│   └────────────────────────────┘     │
│                                      │
│   [headline — 본문 설명 한 줄]       │
│  [logo/handle]                       │
└──────────────────────────────────────┘
   배경: primary, 코드 블록 배경: #0D1117 (고정)
```

- 최대 8줄, 12줄 초과 시 경고
- 라인 번호 회색 (`#6e7681`)
- 언어별 구문 강조: highlight.js `github-dark` 테마

## 카드 번호 인덱싱 (index_label)

- 포맷: `"1/8"` — 분자=현재 카드, 분모=총 장수
- 우상단, 살짝 투명도(alpha 0.7)
- Card N=마지막 카드는 인덱스 생략하고 로고만 키우는 variant 허용

## 훅·CTA 규칙

### Card 1 (hook)
- 헤드라인이 **시각의 중심**. index_label보다도 우위
- 배경에 그래픽 요소(숫자·아이콘·이미지) 크게
- 다음 카드로의 암시: 본문 마지막에 `→` 기호 또는 "이어서 →"

### Card N (cta)
- 액션 동사 1개만: "저장", "DM", "클릭"
- 시리즈 로고/핸들 **1.5배** 크기
- 후킹 질문이나 "다음 시리즈 예고" 선택

## 캡션 작성 가이드 (caption.txt)

1. **첫 125자**가 피드에서 잘리지 않는 "더보기" 앞 영역 — 핵심 주장으로 시작
2. 문단 사이 빈 줄로 리듬 확보
3. 마지막 1~2줄에 CTA 반복 ("저장하고 다시 보세요 📌")
4. 해시태그는 **캡션 끝** 빈 줄 뒤에 몰아서 배치 (포스트 모양 깔끔)
5. 해시태그 6~10개 권장, 대형·중형·니치 믹스

## 접근성

- 본문 대비비 4.5:1 이상 (WCAG AA). photo-overlay에서 그라데이션으로 보장
- 헤드라인 대비비 3:1 이상
- `font-weight: 700` 헤드라인은 대비비 요구가 낮음(3:1)을 활용
- 캡션에 대체 설명(alt) 의식 — 이미지에 없는 맥락을 캡션 첫 문장에 포함
